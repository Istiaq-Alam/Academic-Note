\section{Abstract}
This report presents the design and implementation of a 4-bit Arithmetic Logical Unit (ALU) using basic logic gates in Proteus simulation. 
The ALU performs six operations, both arithmetic and logical, based on select inputs and a carry-in signal. 
Additionally, status flags such as Sign Flag (SF), Carry Flag (CF), Overflow Flag (OF), and Zero Flag (ZF) are integrated to indicate result conditions.

\section{Introduction}
An Arithmetic Logical Unit (ALU) is a fundamental component of a CPU that performs arithmetic and logical operations. 
In this project, a 4-bit ALU was designed to perform operations such as increment, transfer, subtraction, XOR, and XNOR based on control signals.

\section{Truth Table}
The ALU uses three select lines ($S_2, S_1, S_0$) and one carry-in input ($Cin$) to select the required operation. 
The complete truth table is shown below:

\begin{center}
\renewcommand{\arraystretch}{1.4}
\begin{tabular}{|c|c|c|c|c|c|l|}
\hline
\textbf{S2} & \textbf{S1} & \textbf{S0} & \textbf{Cin} & \textbf{X} & \textbf{Y} & \textbf{Function} \\ \hline
0 & 0 & 0 & 1 & 0 & B & Increment B $\rightarrow$ B+1 \\ \hline
0 & 0 & 1 & 0 & 0 & B & Transfer B $\rightarrow$ B \\ \hline
0 & 1 & 0 & 1 & A & B' & Subtraction $\rightarrow$ A - B \\ \hline
0 & 1 & 1 & 0 & A & B' & Subtraction with Borrow $\rightarrow$ A - B - 1 \\ \hline
1 & 0 & 0 & x & A & B & Logical XOR $\rightarrow$ A $\oplus$ B \\ \hline
1 & 0 & 1 & x & A & B' & Logical XNOR $\rightarrow$ A $\odot$ B \\ \hline
\end{tabular}
\end{center}

\section{Function Derivations}
The internal signals X, Y, and Z are defined using the select inputs as follows: \\

\begin{itemize}
    \item $X = A \cdot (S_1 \oplus S_2)$
    \item $Y = B \oplus (S_1 + (S_2 \cdot S_0))$
    \item $Z = \overline{S_2} \cdot C_{in}$
\end{itemize}


These functions determine how the ALU inputs are modified for each operation before being processed by the full adder network.

\section{Status Flags Implementation}
The ALU generates four status flags to indicate the condition of the result:

\begin{itemize}
    \item \textbf{Sign Flag (SF):} Indicates the sign of the result. It is set if the MSB of the result (F3) = 1.
    \[
    SF = F_3
    \]
    
    \item \textbf{Zero Flag (ZF):} Set when all output bits are 0.
    \[
    ZF = \overline{(F_0 + F_1 + F_2 + F_3)}
    \]
    
    \item \textbf{Carry Flag (CF):} Indicates if there is a carry out from the most significant bit.
    \[
    CF = \overline{S_2} \cdot C_{out}
    \]
    
    \item \textbf{Overflow Flag (OF):} Set if there is a signed overflow.
    \[
    OF = (C_3 \oplus C_{out}) \cdot \overline{S_2}
    \]
\end{itemize}

In Proteus, these flags can be displayed using Logicprobe connected to the corresponding outputs of logic circuits:
\begin{itemize}
    \item Use XOR gate for OF
    \item Use NOT and OR gates for ZF
    \item Connect MSB directly for SF
    \item Use the carry output and previous carry from adder for CF
\end{itemize}

\section{Conclusion}
The designed 4-bit ALU successfully performs both arithmetic and logic operations as expected based on select input combinations. 
The inclusion of four status flags allows the ALU to be extended easily for microprocessor-like control applications. 
This experiment provides a clear understanding of how arithmetic logic units function within CPU architecture.
